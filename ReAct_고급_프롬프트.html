📋 ReAct 고급 프롬프트 복사

아래 버튼을 눌러 프롬프트를 복사하세요.
.

우리 시에서 운영 중인 스마트시티 통합플랫폼의 시민 이용률이 
지난 6개월간 계속 감소하고 있습니다. 
현재 월평균 이용자 수는 12,000명 수준이며, 시민 만족도 조사 결과 
'사용이 어렵다'는 응답이 45%입니다. 이용률 개선 방안을 제시해주세요.

이 문제를 해결하기 위해 **ReAct (Reasoning + Acting) 방식**으로
추론과 행동을 반복하며 단계적으로 접근해주세요.

각 사이클마다 [생각 → 행동 → 관찰 → 반성] 패턴을 따라주세요.

═══════════════════════════════════════════════════════════
【CYCLE 1: 초기 상황 분석】
═══════════════════════════════════════════════════════════

💭 **Thought-1 (생각):**
현재 주어진 정보만으로는 구체적인 원인을 특정하기 어렵습니다.
이용률 감소의 정확한 원인을 파악하기 위해 추가 데이터가 필요합니다.
어떤 정보를 먼저 확인해야 할까요?

⚡ **Action-1 (행동):**
다음 데이터를 수집/분석하겠습니다:
→ [행동 A] 월별 이용자 수 추이 데이터 확인 (6개월 상세)
→ [행동 B] 사용자 로그 분석 (어느 기능에서 이탈하는가?)
→ [행동 C] 연령대별 이용률 분포 확인
→ [행동 D] 경쟁 플랫폼(카카오톡, 네이버) 사용 패턴 조사

📊 **Observation-1 (관찰):**
[여기서 실제 데이터를 확인했다고 가정하고 분석 결과를 작성해주세요]

예상 발견 사항:
- 1월: 15,000명 → 6월: 12,000명 (20% 감소)
- 50대 이상 이용자가 60% 이탈
- 로그인 후 평균 3분 내 이탈
- 메인 화면에서 원하는 메뉴를 찾지 못하고 종료하는 비율 65%

🔄 **Reflection-1 (반성):**
관찰 결과, 단순히 "어렵다"는 응답보다 구체적인 문제가 드러났습니다.
특히 50대 이상의 높은 이탈률과 메뉴 탐색 실패가 핵심입니다.
그러나 아직 "왜 메뉴를 찾지 못하는가"에 대한 근본 원인이 불분명합니다.
→ 다음 사이클에서 사용성 테스트가 필요합니다.

═══════════════════════════════════════════════════════════
【CYCLE 2: 근본 원인 탐색】
═══════════════════════════════════════════════════════════

💭 **Thought-2 (생각):**
Cycle 1에서 메뉴 탐색 실패가 주요 문제로 확인되었습니다.
이제 "왜" 메뉴를 찾지 못하는지 구체적 원인을 파악해야 합니다.
실제 사용자 관찰과 비교 분석이 필요합니다.

⚡ **Action-2 (행동):**
→ [행동 A] 50대 이상 시민 10명 대상 사용성 테스트 실시
   (화면 녹화 + 사용자 인터뷰)
→ [행동 B] 현재 UI와 타 지자체 우수 사례 UI 비교 분석
   (서울시, 부산시, 성남시 플랫폼)
→ [행동 C] 현재 메뉴 구조 분석 (깊이, 카테고리 수, 명칭)
→ [행동 D] 웹 접근성 자동 진단 도구 실행

📊 **Observation-2 (관찰):**
[사용성 테스트 결과 작성]

발견된 구체적 문제:
- 메인 화면에 메뉴가 27개나 나열되어 있음 (인지 과부하)
- 행정 용어 사용 (예: "민원24", "통합예약시스템" 등 이해 어려움)
- 아이콘과 텍스트 불일치 (사용자 혼란)
- 글자 크기 12px로 50대 이상이 읽기 어려움
- 우수 사례(서울시)는 5개 대분류 + 2단계 구조로 단순화
- 접근성 진단 결과: WCAG 2.1 기준 40점/100점

🔄 **Reflection-2 (반성):**
문제의 핵심이 명확해졌습니다: 
"정보 과부하 + 직관성 부족 + 접근성 미흡"
그런데 이 모든 것을 한꺼번에 개선하는 것은 비현실적입니다.
어떤 것을 우선 개선해야 가장 빠른 효과를 볼 수 있을까?
→ 다음 사이클에서 우선순위 결정과 실험 설계가 필요합니다.

═══════════════════════════════════════════════════════════
【CYCLE 3: 솔루션 우선순위 결정】
═══════════════════════════════════════════════════════════

💭 **Thought-3 (생각):**
Cycle 2에서 3가지 주요 문제(정보 과부하, 직관성, 접근성)를 발견했습니다.
예산과 시간 제약을 고려하면, 가장 적은 비용으로 가장 큰 효과를 낼 수 있는
"퀵윈(Quick Win)" 전략이 필요합니다. 각 개선안의 효과를 예측해야 합니다.

⚡ **Action-3 (행동):**
→ [행동 A] 3가지 개선안의 비용/효과 분석
   • 안 A: 메뉴 구조 단순화 (27개 → 5개 대분류)
     예상 비용: 2,000만원, 기간: 1개월
   • 안 B: 글자 크기 및 접근성 개선
     예상 비용: 500만원, 기간: 2주
   • 안 C: 행정 용어를 생활 용어로 전환
     예상 비용: 300만원, 기간: 1주

→ [행동 B] 각 개선안의 예상 효과 시뮬레이션
   (A/B 테스트 프로토타입 제작)

→ [행동 C] 소규모 파일럿 테스트 실시
   각 개선안별로 시민 30명 대상 사용성 측정

📊 **Observation-3 (관찰):**
[파일럿 테스트 결과]

- 안 A (메뉴 단순화): 메뉴 탐색 성공률 65% → 88% (↑35%)
- 안 B (접근성 개선): 50대 이상 만족도 45% → 62% (↑38%)
- 안 C (용어 개선): 첫 방문자 이해도 52% → 79% (↑52%)

**예상치 못한 발견:**
안 C(용어 개선)가 가장 저비용이면서도 가장 큰 효과!
특히 안 A + 안 C 조합 시 시너지 효과로 만족도 91%까지 상승

🔄 **Reflection-3 (반성):**
초기에는 UI 리뉴얼(안 A)을 우선시했지만, 
테스트 결과 용어 개선(안 C)이 가장 효과적임을 확인했습니다.
이는 기술적 문제보다 "소통의 문제"였다는 반증입니다.
→ 전략 수정: 안 C 먼저 실행 → 안 A → 안 B 순서로 진행

【CYCLE 4: 실행 전략 구체화】
═══════════════════════════════════════════════════════════

💭 **Thought-4 (생각):**
Cycle 3에서 최적 조합(C→A→B)을 발견했습니다.
이제 실행 가능한 구체적 액션플랜이 필요합니다.
특히 각 단계마다 효과를 측정하고 다음 단계로 진행 여부를 결정해야 합니다.

⚡ **Action-4 (행동):**
→ [행동 A] Phase 1 실행 계획 수립 (안 C: 용어 개선)

   Week 1: 
   - 전 부서 협조 요청 (각 부서에서 사용하는 27개 메뉴 용어 조사)
   - 시민 패널 30명 구성 (연령대별 균등 배분)
   
   Week 2:
   - 행정 용어 → 생활 용어 매핑 테이블 작성
   - 시민 패널 검증 (이해도 테스트)
   
   Week 3:
   - 개발팀 용어 변경 작업
   - 베타 버전 배포 (현재 이용자 10% 대상)
   
   Week 4:
   - A/B 테스트 결과 분석
   - 만족도 5% 이상 개선 시 → 전체 적용 + Phase 2 진행
   - 만족도 개선 미미 시 → 원인 재분석

→ [행동 B] KPI 설정 및 모니터링 체계 구축
   • 일일 모니터링 지표:
     - 신규 가입자 수
     - 평균 체류 시간
     - 메뉴 탐색 성공률
   
   • 주간 모니터링 지표:
     - 주간 활성 사용자(WAU)
     - 연령대별 이용률 변화
     - 고객센터 문의 건수 변화
   
   • 월간 모니터링 지표:
     - 월간 활성 사용자(MAU)
     - 시민 만족도 설문
     - NPS(Net Promoter Score)

→ [행동 C] 리스크 대응 계획 수립
   Risk 1: 용어 변경으로 기존 사용자 혼란
   → 대응: 2주간 병기 표시 (예: "민원24(생활불편신고)")
   
   Risk 2: 부서 간 용어 합의 지연
   → 대응: 시장님 결재로 TF 구성, 의사결정 권한 부여
   
   Risk 3: 개발 리소스 부족
   → 대응: 외부 파트너사 긴급 투입 (예비비 활용)

📊 **Observation-4 (관찰):**
[실행 계획 검증]

- 관련 부서 의견 수렴 결과:
  - 찬성 7개 부서, 유보 2개 부서, 반대 1개 부서
  - 주요 우려: "전문성이 떨어져 보일까 우려"
  
- 예산 확보 현황:
  - Phase 1(용어 개선): 300만원 → 즉시 집행 가능
  - Phase 2(메뉴 단순화): 2,000만원 → 추경 필요
  - Phase 3(접근성): 500만원 → 예비비 활용 가능

- 기술적 검토 결과:
  - 용어 변경: 난이도 하, 2주 소요
  - 메뉴 재구조화: 난이도 중, 4주 소요
  - 접근성 개선: 난이도 하, 2주 소요

🔄 **Reflection-4 (반성):**
실행 계획은 구체화되었으나, 한 가지 놓친 점이 있습니다.
"변화 관리"입니다. 부서 간 이견과 기존 사용자 혼란을 최소화하려면
단순히 시스템만 바꾸는 게 아니라 "변화에 대한 공감대" 형성이 필요합니다.
→ 다음 사이클에서 커뮤니케이션 전략을 보완해야 합니다.

【CYCLE 5: 변화 관리 및 최종 실행안】
═══════════════════════════════════════════════════════════

💭 **Thought-5 (생각):**
Cycle 4에서 기술적 실행 계획은 완성되었지만,
조직 내부 저항과 시민 적응 문제가 남았습니다.
성공적인 변화를 위해서는 "왜 바꾸는가"에 대한 스토리텔링과
단계적 적응 지원이 필수입니다.

⚡ **Action-5 (행동):**
→ [행동 A] 내부 커뮤니케이션 전략

   • 전 직원 대상 타운홀 미팅 (1시간)
     - 현재 문제 공유 (영상으로 시민 불편 사례 보여주기)
     - 개선 효과 시뮬레이션 (Before/After 비교)
     - 각 부서의 기여 포인트 명확화
   
   • 반대 부서 개별 미팅
     - 우려사항 경청 및 대안 제시
     - 파일럿 결과 데이터로 설득
   
   • 월간 진행 상황 공유 (대시보드 구축)

→ [행동 B] 시민 대상 커뮤니케이션 전략

   • Phase 1 시작 1주 전:
     - 공지사항 게시 + 카카오톡 알림
     - "더 쉬워진 스마트시티 플랫폼" 캠페인 시작
     - 변경 내용 안내 영상 제작 (90초 분량)
   
   • Phase 1 진행 중:
     - 튜토리얼 팝업 제공 (첫 로그인 시)
     - 고객센터 상담 인력 2배 증원 (2주간)
     - 실시간 피드백 채널 운영
   
   • Phase 1 완료 후:
     - 개선 효과 공유 (블로그, SNS)
     - 참여 시민 감사 이벤트

→ [행동 C] 적응형 실행 로드맵 (Adaptive Execution)

   ┌─────────────────────────────────────────────┐

   │    ├─ YES → 계속 진행                       │

📊 **Observation-5 (관찰):**
[최종 실행안 검증]

- 이해관계자 분석:
  ✅ 시민: 90% 긍정적 (간담회 결과)
  ✅ 직원: 80% 협조 의지 (타운홀 이후)
  ⚠️ 예산 부서: 2,800만원 승인 (단, 효과 검증 조건부)

  - 3개월 후 이용률: 12,000명 → 16,000명 (33% 증가)
  - 6개월 후 만족도: 55% → 75% (20%p 증가)
  - ROI: 투입 비용 대비 시민 편익 약 5배 추정

- 실패 시나리오 및 대응:
  - Phase 1 실패 시: 원점 복구 후 재기획 (손실 300만원)
  - Phase 2 실패 시: Phase 1 성과 유지, Phase 3 선실행
  - 전체 실패 시: 외부 전문가 자문 + 근본적 재설계

🔄 **Reflection-5 (반성):**
5번의 사이클을 거쳐 다음을 달성했습니다:
1. 데이터 기반 문제 진단 ✅
2. 구체적 원인 규명 ✅
3. 실험적 검증을 통한 솔루션 선정 ✅
4. 실행 가능한 단계적 계획 ✅
5. 변화 관리 및 리스크 대응 ✅

**중요한 인사이트:**
- 초기 가정("기술적 문제")이 틀렸고, 실제로는 "소통의 문제"였음
- 최소 비용 솔루션(용어 개선)이 최대 효과를 냄
