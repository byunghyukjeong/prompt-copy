## 🛠️ [개발 요청 프롬프트] 대전 서구 체육시설 찾기 플랫폼 ### 👩‍💻 역할 (Role) 당신은 **React 기반 프론트엔드 개발 전문가**입니다. 다음 기술 스택에 익숙하다고 가정합니다: - Next.js, React Query, Zustand 또는 Jotai - Tailwind CSS, TypeScript - 카카오맵 또는 네이버지도 API --- ### 🗂️ 데이터 및 환경 (Context) - **데이터 출처**: 공공데이터포털의 대전 서구 공공 체육시설 CSV 파일 - **주요 필드**: - 🏢 시설명 - 📌 유형 - 🗺️ 위치좌표 (위도, 경도) - ⏰ 운영시간 - 💰 이용요금 - ☎️ 연락처 - 🚻 편의시설 (화장실, 주차 등) - **사용자 대상**: - 👪 주민 - 🧑‍💼 체육시설 관리자 - 🧳 관광객 --- ### 🎯 목적 (Goal) React 기반 플랫폼에서 아래 목표를 달성해 주세요: - ✅ 사용자가 체육시설을 **쉽고 빠르게 검색/조회**할 수 있도록 구현 - ✅ 지도 연동을 통해 시설 위치를 시각적으로 표시 - ✅ 다양한 기준(이름, 거리, 운영시간 등)으로 필터링 - ✅ 즐겨찾기 및 리뷰 기능 추가 - ✅ 모바일과 데스크탑에서 모두 보기 좋게 만들기 --- ### 💡 주요 기능 목록 (Feature List) | 기능 | 설명 | |------|------| | 🗺️ 지도 기반 시각화 | 카카오맵 또는 네이버지도 API로 위치 표시 | | 🔍 검색/필터 기능 | 이름, 유형, 거리, 운영시간 등으로 필터링 | | 📋 리스트/카드 뷰 전환 | 보기 방식 선택 가능 | | 📄 상세정보 페이지 | 운영시간, 연락처, 요금 등 세부정보 표시 | | ⭐ 즐겨찾기 | 로컬스토리지 또는 Mock API로 구현 | | 💬 리뷰 기능 | 간단한 리뷰/평점 기능 (로컬 또는 임시 서버) --- ### 🧠 Chain of Thought 방식 개발 지침 각 기능을 다음과 같은 순서로 개발해 주세요: 1. **문제 정의** - "이 기능은 어떤 문제를 해결하는가?"를 한 문장으로 설명 2. **데이터/상태 구조 설계** - 필요한 타입, 상태, 컴포넌트 구조를 정의 3. **의사코드 또는 주석 작성** - 복잡한 부분은 pseudocode나 상세 주석 작성 4. **단계별 구현** - 기능별로 분할 → 구현 → 통합 순으로 작업 5. **중간 점검** - 각 단계 완료 시 코드 + UI 결과를 확인할 수 있도록 출력 6. **상태관리/API 연동도 사고 흐름에 따라 작성** - 예: 즐겨찾기 상태는 Zustand로 관리, API는 React Query로 연동 등 --- ### 🧩 예시 코드 시작점 ts // 체육시설 인터페이스 정의 예시 interface Facility { id: string; name: string; type: string; latitude: number; longitude: number; hours: string; fee: string; contact: string; amenities: string[]; } --- ### 🙋‍♂️ 참고 및 추가 요청 예시 - "CSV 데이터를 React에서 어떻게 불러오나요?" - "Zustand로 즐겨찾기를 어떻게 저장하죠?" - "Tailwind로 모바일/PC 반응형을 동시에 만들려면?" --- 이 요청에 따라 단계별로 **코드 + 설명**을 포함한 결과를 생성해 주세요. 필요 시, 각 단계별 코드와 함께 상세 설명도 출력해 주세요.
